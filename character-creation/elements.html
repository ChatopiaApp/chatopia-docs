Character Elements & Limits

Every AI character on Chatopia consists of five core elements with specific character limits. Understanding these elements and their constraints is essential for creating functional characters that become NFTs.

Character NFT Structure​

Digital Ownership Basics​

Characters exist as NFTs with two key aspects:

  • Character Limits: Maximum text allowed for each element (determined by your NFT)
  • Character Content: The actual personality design you create within those limits
  • Permanent Ownership: Characters cannot be deleted or removed by the platform

Starting Capabilities​

All newly minted characters begin with identical baseline limits and performance stats:

  • Content Limits: Fixed character allowances for each design element
  • Performance Stats: Functional capabilities affecting character operation
  • Upgradeable: All limits can be enhanced through training after minting

The Five Core Elements​

Every character requires these essential components in order:

1. Name​

Starting Limit: 15 characters

Function:

  • Character identifier and primary branding
  • Includes spaces, punctuation, and special characters
  • Used across platform for character recognition

Examples:

  • "Alex Chen" (9 characters)
  • "Luna Star" (9 characters)
  • "Dr. Morgan" (10 characters)

2. Tagline​

Starting Limit: 30 characters

Function:

  • Brief catchphrase or motto representing character voice
  • Used in character marketing and discovery
  • First impression alongside character name

Examples:

  • "Knowledge is power." (18 characters)
  • "Every challenge matters." (22 characters)
  • "Creativity sparks magic." (22 characters)

3. Description​

Starting Limit: 200 characters

Function:

  • Character background and personality overview
  • User's first detailed understanding of character capabilities
  • Foundation for user expectations before interaction

Example: "A wise mentor with 20 years of experience guiding others. Known for patience, practical advice, and seeing potential in everyone. Specializes in problem-solving and growth." (186 characters)

4. Greeting​

Starting Limit: 500 characters

Function:

  • First message users see when starting conversations
  • Sets tone and expectations for character interactions
  • Immediate demonstration of character personality

Examples:

  • "Hello! I'm Alex, and I love helping people solve challenging problems. What's on your mind today that we could work through together?"
  • "Welcome, friend! Luna here, ready to explore new ideas and adventures. What fascinating topic shall we discuss?"

5. Definition​

Starting Limit: 500 characters

Function:

  • Detailed personality framework guiding all character responses
  • Behavioral guidelines and response patterns for AI system
  • Most comprehensive element determining character consistency

Examples:

Structured Format:

{{char}} = ["patient mentor" + "asks thoughtful questions" + "warm tone" + "problem-solving expert" + "encouraging"]

Free-form Description:

Alex is a patient mentor who always asks thoughtful questions before giving advice. Speaks in a warm, encouraging tone. Specializes in breaking down complex problems into manageable steps.

Dialogue Examples:

User: "I'm feeling overwhelmed."
{{char}}: "That sounds challenging. What specifically feels most overwhelming right now?"

Personality Instructions:

- Always respond with empathy first
- Ask clarifying questions
- Provide practical advice
- Use encouraging language

Character Performance Stats​

Beyond content elements, characters have functional capabilities:

Memory Depth​

Starting Value: 10 past messages retained

Function:

  • Number of previous messages character remembers in conversations
  • Enables reference to earlier conversation parts
  • Maintains conversation continuity and coherence

Concurrent Users​

Starting Value: 100 simultaneous users supported

Function:

  • Maximum users who can chat with character simultaneously
  • Eliminates waiting times for character access
  • Direct multiplier for interaction capacity

Daily Messages​

Starting Value: 50 free messages per user per day

Function:

  • Free message allowance per user before restrictions
  • Balances user accessibility with platform monetization
  • Affects user engagement depth and retention

These stats can be improved through training to enhance performance and earning potential.

Character Design Requirements​

Content Guidelines​

All character elements must comply with platform standards:

  • Appropriate Content: All elements suitable for platform audience
  • No Harmful Content: Cannot contain offensive, illegal, or harmful material
  • Community Guidelines: Must follow platform terms of service
  • Intellectual Property: Respect copyright and trademark laws

Technical Constraints​

Character creation operates within specific technical limitations:

  • Current Limit Enforcement: Cannot exceed your character's current limits for any element
  • Element Completion: All five elements required for minting
  • Content Processing: Elements processed by AI system for character functionality

Character Updates​

Characters can be modified after minting:

  • Content Editing: All elements can be updated within current character limits
  • Training Enhancement: Character limits can be increased through training
  • Personality Refinement: Characters can be improved and refined over time

Limit Enhancement Through Training​

Training System Integration​

Character limits can be expanded after minting:

  • Random Training: Training randomly selects one stat to improve
  • Token Cost: CHATOPIA tokens required for each training session
  • Permanent Improvements: All training enhancements are permanent

Enhanced Capabilities​

Training can improve both content limits and performance stats:

Content Limit Improvements:

  • Name Length: 15 → 30 characters maximum
  • Tagline Length: 30 → 50 characters maximum
  • Description Length: 200 → 500 characters maximum
  • Greeting Length: 500 → 2,000 characters maximum
  • Definition Length: 500 → 30,000 characters maximum

Performance Stat Improvements:

  • Memory Depth: 10 → 50 past messages retained
  • Concurrent Users: 100 → 1,000,000 simultaneous users supported
  • Daily Messages: 50 → 200 free messages per user per day
